## The Log: What You're Seeing

**The Error:**

```bash
[INFO] Starting OpenClaw...
[ERROR] Error: Cannot find module '@mariozechner/clipboard-linux-arm-gnueabihf'
Require stack:
- /app/dist/index.js:15:23
- at processTicksAndRejections (node:internal/process/task_queues:96:5)

[System] Clipboard module failed to load
[Agent] Unable to use clipboard functionality
```

**Context:** This error occurs on Linux ARM devices (Raspberry Pi, ARM servers, etc.) when OpenClaw tries to load the clipboard package. The package has prebuilt binaries for x86/x64 but not for ARM.

**Observed Behavior:** OpenClaw starts but crashes or loses clipboard functionality. Copy/paste operations may fail silently.

---

## Why This Happens (It's an Architecture Mismatch)

This is a **Prebuilt Binary Availability Issue**.

OpenClaw uses the `@mariozechner/clipboard` package for clipboard operations. This package includes:
- **Prebuilt native binaries** for x86/x64 (Intel/AMD)
- **Fallback pure-JS implementation** for other architectures

On Linux ARM (aarch64), if the npm version is old (< 9.0.0), it may:
- Try to load a non-existent ARM binary
- Fail to fall back to the pure-JS implementation
- Crash with "Cannot find module" error

### The ARM Architecture Problem

| Architecture | Prebuilt Binary | Pure-JS Fallback | Status |
|--------------|-----------------|-------------------|--------|
| **x86/x64** (Intel/AMD) | ✅ Included | ✅ Available | Works out of box |
| **ARM64** (Raspberry Pi, ARM servers) | ❌ Not included | ⚠️ May not load | Needs npm v9+ |
| **ARMv7** (Raspberry Pi Zero/older) | ❌ Not included | ❌ Not compatible | Needs manual build |

## Solution A: Upgrade npm (The Fix)

The easiest fix is to upgrade npm to version 9.0.0 or later, which includes ARM prebuilds.

### Step 1: Check Current npm Version

```bash
# Check npm version
npm --version

# If below 9.0.0, upgrade needed
# Output example: v8.19.4 (too old)
```

### Step 2: Upgrade npm

```bash
# Using npm itself
npm install -g npm@latest

# Or using Node.js installer
# Download latest LTS from https://nodejs.org/
```

### Step 3: Clear npm Cache and Reinstall

```bash
# Clear npm cache
npm cache clean --force

# Remove node_modules
rm -rf node_modules package-lock.json

# Reinstall OpenClaw
npm install @m1heng/openclaw

# Or using npx
npx openclaw start
```

### Step 4: Verify Clipboard Module Loads

```bash
# Test if clipboard works
node -e "require('@mariozechner/clipboardy'); console.log('Clipboard OK');"

# Should output: Clipboard OK
```

## Solution B: Use Pure-JS Clipboard Fallback

If upgrading npm doesn't work or you're stuck on an old Node version, use the fallback clipboard package.

### Step 1: Install Clipboardy (Pure-JS)

```bash
# Clipboardy is the pure-JS fallback, works on any architecture
npm install clipboardy
```

### Step 2: Configure OpenClaw to Use Clipboardy

```javascript
// In your OpenClaw config
const clipboardy = require('clipboardy');

// Use clipboardy instead of @mariozechner/clipboard
clipboardy.writeSync('Your text here');

const text = clipboardy.readSync();
```

### Step 3: Update OpenClaw Config

```javascript
// openclaw_config.json
{
  "clipboardModule": "clipboardy",
  "clipboardOptions": {
    "copyWithoutFormat": true
  }
}
```

## Solution C: Build from Source (Last Resort)

If prebuilts don't exist for your ARM version, compile from source.

### Step 1: Install Build Tools

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y build-essential python3

# Alpine Linux
apk add build-base python3
```

### Step 2: Build Module

```bash
# Clone the clipboard repository
git clone https://github.com/mariozechner/clipboardy.git
cd clipboardy

# Build for ARM
npm rebuild
npm install

# Install globally
npm install -g .
```

### Step 3: Verify Installation

```bash
node -e "require('clipboardy'); console.log('Clipboardy OK');"
```
